# Generated-code classifier

`IGeneratedCodeReport` separates generated artefacts from hand-written code.
Read-only; deterministic. Schema: `sharkcraft.generated-code/v1`. Module:
`packages/inspector/src/generated-code.ts`.

## Signals

The classifier looks at:

1. **Directory hints** — `dist/`, `build/`, `out/`, `.next/`, `.nuxt/`,
   `.angular/`, `.svelte-kit/`, `coverage/`, `__snapshots__/`, `generated/`,
   `__generated__/`, `gen/`, `proto/`, `node_modules/`, `.nx/cache/`.
2. **Filename heuristics** — lockfiles (`bun.lockb`, `pnpm-lock.yaml`,
   `Cargo.lock`, …), Angular env files (`src/environments/environment.*`),
   compiled `.d.ts` headers.
3. **Banner markers** in the first ~30 lines of source files:
   - `@generated`
   - `DO NOT EDIT`
   - `auto-generated`
   - `Code generated by … DO NOT EDIT`
   - `openapi-generator`, `swagger-codegen`
   - `graphql-code-generator`
   - `protoc-gen-…`
   - `prisma-client-js`

## Output

```ts
interface IGeneratedCodeReport {
  generatedFiles: IGeneratedFileEntry[];     // path + kind + confidence + reason + markerLine
  handwrittenFiles: string[];
  mixedFiles: string[];                      // reserved
  generatedRoots: IGeneratedRoot[];          // dir-level rollups with file counts
  protectedRules: IProtectedRule[];          // ready-to-paste rule entries
  recommendedPolicyRules: IRecommendedGeneratedPolicy[];
  filesScanned: number;
  limitations: string[];
}
```

## CLI

```bash
shrk generated report                      # text
shrk generated report --format markdown
shrk generated report --format json
shrk generated protect --write-drafts      # writes sharkcraft/ingestion/GENERATED_PROTECT.md
```

`generated protect` writes only under `sharkcraft/ingestion/`. It does
not modify live policies.

## MCP

`get_generated_code_report` — returns the structured report + a
next-command hint. Never writes.

## Recommended policy

When generated roots are detected, the ingest pipeline surfaces a
`policy.generated-readonly` recommendation: plans that modify files under a
generated root require an explicit `generated` intent on the agent contract,
otherwise the plan is rejected at review. Adopt it via
`sharkcraft/policies.ts`.
